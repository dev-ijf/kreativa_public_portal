## Overview

Bank Muamalat Indonesia (BMI) calls **your** backend service over HTTPS POST. Your system acts as the **mitra (partner) backend**, receiving requests from BMI, processing them against your internal database, and returning a JSON response ŌĆö all wrapped in JWT (HS256) encryption.

```
Client Apps  ŌåÉŌåÆ  BMI System  ŌöĆŌöĆHTTPS POSTŌöĆŌöĆŌ¢║  Your Mitra Backend (Next.js on Vercel)
```

### Communication Specs

| Property | Value |
|---|---|
| Protocol | HTTPS POST |
| Content-Type Header | `text/plain; charset=ISO-8859-1` |
| Data Format | JSON |
| Encryption | JWT (HS256) |
| Min TLS Version | 1.2 |

### Virtual Account Number Format

```
XXXX  +  XX  +  XXXXXXXXXX
Ōöé         Ōöé      ŌööŌöĆ 10 digits: unique identifier generated by mitra
Ōöé         ŌööŌöĆ 2 digits: product code (e.g., 00=tuition fee, 01=semester, 02=lab fee)
ŌööŌöĆ 4 digits: BIN number assigned by BMI
```

Total: **16 digits**

## Shared Utilities

### `/lib/va/jwt.ts`

```typescript
import * as jose from "jose";

const SECRET = new TextEncoder().encode(process.env.BMI_JWT_SECRET!);

/**
 * Decode and verify a JWT token sent by BMI.
 * BMI sends the body as a JWT string (text/plain).
 */
export async function decodeToken(token: string): Promise<Record<string, unknown>> {
  const { payload } = await jose.jwtVerify(token, SECRET, {
    algorithms: ["HS256"],
  });
  return payload as Record<string, unknown>;
}

/**
 * Encode your JSON response as a JWT for sending back to BMI.
 */
export async function encodeToken(payload: Record<string, unknown>): Promise<string> {
  return new jose.SignJWT(payload)
    .setProtectedHeader({ alg: "HS256" })
    .sign(SECRET);
}
```

### `/lib/va/response.ts`

```typescript
import { encodeToken } from "./jwt";

/**
 * Build and return a JWT-wrapped response with Content-Type: text/plain
 */
export async function buildResponse(
  payload: Record<string, unknown>,
  status = 200
): Promise<Response> {
  const token = await encodeToken(payload);
  return new Response(token, {
    status,
    headers: {
      "Content-Type": "text/plain; charset=ISO-8859-1",
    },
  });
}
```

### `/lib/va/validate.ts`

```typescript
/**
 * Validate mitra credentials from the decoded JWT payload.
 * Returns true if credentials match (or if mitra doesn't use credentials).
 */
export function validateCredentials(
  username?: string,
  password?: string
): boolean {
  const expectedUser = process.env.MITRA_USERNAME;
  const expectedPass = process.env.MITRA_PASSWORD;

  // If mitra doesn't configure credentials, skip validation
  if (!expectedUser && !expectedPass) return true;

  return username === expectedUser && password === expectedPass;
}

/**
 * Parse VANO: extract BIN (4), product code (2), customer ID (10)
 */
export function parseVANO(vano: string): {
  bin: string;
  productCode: string;
  customerId: string;
} | null {
  if (!vano || vano.length !== 16) return null;
  return {
    bin: vano.slice(0, 4),
    productCode: vano.slice(4, 6),
    customerId: vano.slice(6, 16),
  };
}

/**
 * Format customer name: uppercase, max 30 chars, no symbols
 */
export function formatCustomerName(name: string): string {
  return name
    .toUpperCase()
    .replace(/[^A-Z\s]/g, "")
    .slice(0, 30);
}
```

### 1. `POST /api/va/inquiry`

**Purpose**: BMI sends a virtual account number and asks your system to return the customer's billing information.

#### `/app/api/va/inquiry/route.ts`

```typescript
import { NextRequest } from "next/server";
import { decodeToken } from "@/lib/va/jwt";
import { buildResponse } from "@/lib/va/response";
import { validateCredentials, parseVANO, formatCustomerName } from "@/lib/va/validate";

// Replace this with your actual DB lookup logic
async function getBillingByVANO(vano: string) {
  // Example stub ŌĆö implement with your DB (Neon, Drizzle, etc.)
  // Return null if not found, or { bill, description, description2, custName, isPaid }
  return null;
}

export async function POST(req: NextRequest) {
  let payload: Record<string, unknown>;

  try {
    const body = await req.text();
    payload = await decodeToken(body);
  } catch {
    return buildResponse({ ERR: "55", METHOD: "INQUIRY" });
  }

  const {
    CCY,
    VANO,
    TRXDATE,
    METHOD,
    USERNAME,
    PASSWORD,
    CHANNELID,
    REFNO,
  } = payload as Record<string, string>;

  // Validate credentials
  if (!validateCredentials(USERNAME, PASSWORD)) {
    return buildResponse({ ERR: "55", METHOD: "INQUIRY" });
  }

  // Validate VANO format
  const parsed = parseVANO(VANO);
  if (!parsed) {
    return buildResponse({ ERR: "30", METHOD: "INQUIRY" });
  }

  // Lookup billing data
  const billing = await getBillingByVANO(VANO);

  if (!billing) {
    return buildResponse({ ERR: "15", METHOD: "INQUIRY" });
  }

  if (billing.isPaid) {
    return buildResponse({ ERR: "88", METHOD: "INQUIRY" });
  }

  // Billing amount must be multiplied by 100 (two decimal digits appended)
  const billAmount = billing.bill * 100;

  // Bill cannot be negative; send 0 if negative
  const safeBill = billAmount < 0 ? 0 : billAmount;

  return buildResponse({
    CCY: CCY ?? "360",
    BILL: String(safeBill),
    DESCRIPTION: billing.description.slice(0, 40),
    DESCRIPTION2: billing.description2?.slice(0, 256) ?? "",
    CUSTNAME: formatCustomerName(billing.custName),
    ERR: "00",
    METHOD: "INQUIRY",
  });
}
```

#### Inquiry Field Reference

| Field | Direction | Type | Notes |
|---|---|---|---|
| `CCY` | BMI ŌåÆ Mitra | string(3) | Always `"360"` (IDR) |
| `VANO` | BMI ŌåÆ Mitra | string(16) | 16-digit virtual account number |
| `TRXDATE` | BMI ŌåÆ Mitra | string(14) | Format: `YYYYMMDDHHmmSS` |
| `METHOD` | BMI ŌåÆ Mitra | string | `"INQUIRY"` |
| `USERNAME` | BMI ŌåÆ Mitra | string | Optional mitra verifier |
| `PASSWORD` | BMI ŌåÆ Mitra | string | Optional mitra security key |
| `CHANNELID` | BMI ŌåÆ Mitra | string | `1`=ATM, `2`=Teller, `3`=iBanking, `4`=EDC, `5`=mBanking |
| `REFNO` | BMI ŌåÆ Mitra | string(12) | Reference number from BMI |
| `CCY` | Mitra ŌåÆ BMI | string(3) | Echo back `"360"` |
| `BILL` | Mitra ŌåÆ BMI | string(12,2) | Amount ├Ś 100 (e.g., `"2765000000"`) |
| `DESCRIPTION` | Mitra ŌåÆ BMI | string(40) | Billing description |
| `DESCRIPTION2` | Mitra ŌåÆ BMI | string(256) | Extended billing detail |
| `CUSTNAME` | Mitra ŌåÆ BMI | string(30) | Uppercase, no symbols, max 30 chars |
| `ERR` | Mitra ŌåÆ BMI | string(2) | `"00"` = success |

#### Inquiry Rules

- `CUSTNAME` must be **ALL CAPS**, **no symbols**, truncated to **30 characters**
- `BILL` must never be negative ŌĆö send `0` if negative
- `BILL` must be the actual amount multiplied by **100** before sending
- If VANO not found ŌåÆ respond `ERR: "15"`
- If already paid ŌåÆ respond `ERR: "88"`

### 2. `POST /api/va/payment`

**Purpose**: BMI notifies your system of a successful payment transaction. You must record it and update the billing status.

#### Payment Types

| Type | Rule |
|---|---|
| **Full** | Payment must **exactly equal** the bill. Reject with `"16"` if different |
| **Partial** | Payment can be Ōēż bill. Reject with `"13"` if payment exceeds bill |
| **Open/None** | Any amount is accepted |

#### Unique Key

Every payment transaction is uniquely identified by: **`VANO + REFNO + TRXDATE`**  
Duplicate transactions **must be rejected** with `ERR: "30"`.

#### `/app/api/va/payment/route.ts`

```typescript
import { NextRequest } from "next/server";
import { decodeToken } from "@/lib/va/jwt";
import { buildResponse } from "@/lib/va/response";
import { validateCredentials, parseVANO, formatCustomerName } from "@/lib/va/validate";

type PaymentType = "FULL" | "PARTIAL" | "OPEN";

// Replace these with your actual DB functions
async function getBillingByVANO(vano: string): Promise<{
  bill: number;
  description: string;
  description2: string;
  custName: string;
  isPaid: boolean;
  paymentType: PaymentType;
} | null> {
  return null; // implement with your DB
}

async function isDuplicateTransaction(
  vano: string,
  refno: string,
  trxdate: string
): Promise<boolean> {
  return false; // implement with your DB
}

async function recordPayment(
  vano: string,
  refno: string,
  trxdate: string,
  payment: number
): Promise<boolean> {
  return true; // implement with your DB; return false on failure
}

export async function POST(req: NextRequest) {
  let payload: Record<string, unknown>;

  try {
    const body = await req.text();
    payload = await decodeToken(body);
  } catch {
    return buildResponse({ ERR: "55", METHOD: "PAYMENT" });
  }

  const {
    TRXDATE,
    CCY,
    REFNO,
    BILL,
    PAYMENT,
    CHANNELID,
    VANO,
    CUSTNAME,
    USERNAME,
    PASSWORD,
  } = payload as Record<string, string>;

  // Validate credentials
  if (!validateCredentials(USERNAME, PASSWORD)) {
    return buildResponse({ ERR: "55", METHOD: "PAYMENT" });
  }

  // Validate VANO format
  const parsed = parseVANO(VANO);
  if (!parsed) {
    return buildResponse({ ERR: "30", METHOD: "PAYMENT" });
  }

  // BMI sends BILL and PAYMENT already ├Ś100 ŌĆö divide by 100 before processing
  const billAmount = parseFloat(BILL) / 100;
  const paymentAmount = parseFloat(PAYMENT) / 100;

  // Check for duplicate transaction
  const isDuplicate = await isDuplicateTransaction(VANO, REFNO, TRXDATE);
  if (isDuplicate) {
    return buildResponse({ ERR: "30", METHOD: "PAYMENT" });
  }

  // Fetch billing record
  const billing = await getBillingByVANO(VANO);
  if (!billing) {
    return buildResponse({ ERR: "30", METHOD: "PAYMENT" });
  }

  if (billing.isPaid) {
    return buildResponse({ ERR: "88", METHOD: "PAYMENT" });
  }

  // Payment type validation
  if (billing.paymentType === "FULL" && paymentAmount !== billing.bill) {
    return buildResponse({ ERR: "16", METHOD: "PAYMENT" });
  }

  if (billing.paymentType === "PARTIAL" && paymentAmount > billing.bill) {
    return buildResponse({ ERR: "13", METHOD: "PAYMENT" });
  }

  // Record the payment
  let recorded: boolean;
  try {
    recorded = await recordPayment(VANO, REFNO, TRXDATE, paymentAmount);
  } catch {
    // Timeout or system error ŌĆö return suspect (99), only for Payment
    return buildResponse({ ERR: "99", METHOD: "PAYMENT" });
  }

  if (!recorded) {
    return buildResponse({ ERR: "12", METHOD: "PAYMENT" });
  }

  return buildResponse({
    CCY: CCY ?? "360",
    BILL: billAmount * 100, // return as-received (├Ś100)
    CUSTNAME: formatCustomerName(billing.custName),
    DESCRIPTION: billing.description.slice(0, 40),
    DESCRIPTION2: billing.description2?.slice(0, 256) ?? "",
    ERR: "00",
    METHOD: "PAYMENT",
  });
}
```

| Field | Direction | Type | Notes |
|---|---|---|---|
| `TRXDATE` | BMI ŌåÆ Mitra | string(14) | `YYYYMMDDHHmmSS` |
| `CCY` | BMI ŌåÆ Mitra | string(3) | `"360"` |
| `REFNO` | BMI ŌåÆ Mitra | string(12) | Unique reference from BMI |
| `BILL` | BMI ŌåÆ Mitra | string(12,2) | Bill ├Ś 100 (divide by 100 before use) |
| `PAYMENT` | BMI ŌåÆ Mitra | string(12,2) | Payment ├Ś 100 (divide by 100 before use) |
| `CHANNELID` | BMI ŌåÆ Mitra | string | `1`=ATM, `2`=Teller, `3`=iBanking, `4`=EDC, `5`=mBanking |

### 3. `POST /api/va/signon`

**Purpose**: BMI initiates a session. Transactions are only allowed after Sign On succeeds. Sign On is **performed once** when the system starts ŌĆö it does not repeat unless the server restarts.

#### `/app/api/va/signon/route.ts`

```typescript
import { NextRequest } from "next/server";
import { decodeToken } from "@/lib/va/jwt";
import { buildResponse } from "@/lib/va/response";
import { validateCredentials } from "@/lib/va/validate";

export async function POST(req: NextRequest) {
  let payload: Record<string, unknown>;

  try {
    const body = await req.text();
    payload = await decodeToken(body);
  } catch {
    return buildResponse({ ERR: "55", METHOD: "SIGNON" });
  }

  const { SIGNONINFO, METHOD, USERNAME, PASSWORD } = payload as Record<string, string>;

  if (!validateCredentials(USERNAME, PASSWORD)) {
    return buildResponse({ ERR: "55", METHOD: "SIGNON" });
  }

  if (!SIGNONINFO) {
    return buildResponse({ ERR: "30", METHOD: "SIGNON" });
  }

  // SIGNONINFO format: "YYYYMMDDHHmmSS;bankmuamalatindonesia"
  const [signonDate, authCode] = SIGNONINFO.split(";");

  // Optionally verify authCode is from BMI
  if (authCode !== "bankmuamalatindonesia") {
    return buildResponse({ ERR: "55", METHOD: "SIGNON" });
  }

  const mitraName = process.env.MITRA_NAME ?? "MitraName";

  return buildResponse({
    ERR: `${signonDate};00;${mitraName}`,
    METHOD: "SIGNON",
  });
}
```

#### Sign On Field Reference

| Field | Direction | Type | Notes |
|---|---|---|---|
| `SIGNONINFO` | BMI ŌåÆ Mitra | string | `"YYYYMMDDHHmmSS;bankmuamalatindonesia"` |
| `METHOD` | BMI ŌåÆ Mitra | string | `"SIGNON"` |
| `USERNAME` | BMI ŌåÆ Mitra | string | Optional |
| `PASSWORD` | BMI ŌåÆ Mitra | string | Optional |
| `ERR` | Mitra ŌåÆ BMI | string | Format: `"signondate;00;[mitra_name]"` |

#### Sign On Rules

- Sign On is **one-time only** per server uptime
- Transactions must be rejected until Sign On succeeds
- BMI will not re-send Sign On unless the server restarts

## Response Codes

Your system returns a 2-digit `ERR` code in all responses:

| Code | Action | Description |
|---|---|---|
| `00` | **Approve** | Transaction completed successfully |
| `12` | Declined | Invalid transaction ŌĆö failed to update billing in mitra system |
| `13` | Declined | Invalid amount ŌĆö payment exceeds bill (for Partial type) or below minimum |
| `15` | Declined | Bill ID not found ŌĆö VANO not in mitra database |
| `16` | Declined | Invalid full amount ŌĆö payment does not exactly match bill (for Full type) |
| `30` | Declined | Format error ŌĆö bad/missing field values or wrong message format |
| `55` | Declined | Username / Password / EncryptKey not valid |
| `88` | Declined | Bill already paid, or bill amount is 0 (except for Open/None type) |
| `99` | **Suspect** | Timeout or system error ŌĆö transaction outcome unknown. BMI treats it as success. Mitra should reconcile via VA recon data. **Payment only.** |

### How to Use the Postman Collection

1. **Import** the JSON above into Postman via *File ŌåÆ Import*
2. **Set Collection Variables** (click the collection ŌåÆ *Variables* tab):
   - `BASE_URL` ŌåÆ your Vercel deployment URL
   - `JWT_SECRET` ŌåÆ the shared secret from BMI
   - `MITRA_USERNAME` / `MITRA_PASSWORD` ŌåÆ your configured credentials
   - `MITRA_NAME` ŌåÆ your mitra name
   - `VANO` ŌåÆ a valid 16-digit VA number in your database
3. **Generate JWT Tokens**: Because the body must be a signed JWT, use [https://jwt.io](https://jwt.io) to build each token with your secret, then paste the token string into the corresponding `JWT_TOKEN_*` variable. For automated testing, add a Postman Pre-request Script using the `jsrsasign` library loaded via CDN.
4. **Run requests** in order: Sign On ŌåÆ Inquiry ŌåÆ Payment ŌåÆ Sign Off / Echo

## Notes & Gotchas

- **BILL/PAYMENT multiplier**: BMI always sends amounts ├Ś 100 (two decimal zeros appended). When receiving (Payment), divide by 100. When sending (Inquiry response), multiply by 100.
- **No negative bills**: Always send `0` if the calculated amount is negative.
- **Suspect (99)**: Only valid for Payment. When returned, BMI treats the payment as successful and expects the mitra to reconcile via VA recon report.
- **Sign On is persistent**: Vercel functions are stateless, so you'll need an external store (Redis, DB flag) to track Sign On/Sign Off state across serverless invocations.
- **Duplicate detection**: Store `VANO + REFNO + TRXDATE` in your DB to prevent duplicate payment processing.
- **TLS requirement**: Vercel handles TLS 1.2+ by default ŌĆö no extra configuration needed.
